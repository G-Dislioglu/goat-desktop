# GOAT Desktop Recovery Scan Notes

Updated: 2026-05-28

Use this as the default read-only recovery scan frame.

## Scope

Assess the current GOAT Desktop repo state against:

- `STATE.md`
- `RADAR.md`
- `CLAUDE-CONTEXT.md`
- `GOAT-DESKTOP-VISION.md`
- current tests
- local `/healthz`, if a local bridge is running or relevant

## Must Check

- Current branch, commit, remote, and working tree.
- Latest STATE entry and whether it matches code/tests.
- Stage 1/2/3/4 boundaries.
- `/builder-cue` proposal-only behavior.
- Test status and compile status.
- Any leftover screenshot, audio, or smoke-test artifacts.

## Must Not Do

- Do not start LiveTalk or microphone checks.
- Do not run real screen actions.
- Do not use real user fields.
- Do not connect to production Builder with real secrets.
- Do not modify existing governance docs during a scan.

## Output

Write findings as a short review packet with:

- actual checked commit
- commands run
- findings ordered by severity
- safety regressions, if any
- next 2-4 BP blocks
