# GOAT Desktop App Goal

Updated: 2026-05-28
Baseline: 2faf005 fix(ui): show stage3 popup review path

GOAT Desktop is a local Windows assistant for normal users. It should help the user understand the visible desktop, mark safe targets, and perform only narrow mouse or keyboard actions after local verification and explicit user approval.

Builder is a proposal source, not an execution authority. GOAT Desktop owns the local trust boundary:

- local geometry verification in `broker.py`
- visible popup approval in `tray.py` / `popup.py`
- staged action gating in `action_gate.py`
- narrow executors for Stage 1 and Stage 2 only
- review-only handling for Stage 3
- technical lock for Stage 4

## Product Priorities

1. Make the assistant clear for normal users.
2. Preserve user control before any real mouse or keyboard action.
3. Keep action semantics simple: check target, then either execute safe action or explain why not.
4. Prefer controlled, local, auditable behavior over broad autonomy.
5. Keep LiveTalk secondary unless explicitly requested.

## Non-Goals

- No cloud deployment.
- No trading, purchasing, deleting, sending, saving, deploying, or similar consequential OS action.
- No automatic interaction with secrets, passwords, tokens, 2FA, CVV, PIN, or recovery codes.
- No Vision-only coordinate authority.
- No tests on real user fields.

## Current Safe Action Model

- Stage 1: hover, move, scroll. User approval required. No click, no typing.
- Stage 2: one-line text input. User approval and `safe_text_context=true` required.
- Stage 3: important action review in popup. No OS execution.
- Stage 4: technical lock. User handles manually.
